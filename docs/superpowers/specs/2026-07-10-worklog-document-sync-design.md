# 工作日志、实验数据与项目文档同步整理设计

## 目标

以实际实验脚本和 `outputs/` 产物为事实来源，补齐并统一 `工作日志.md` 的中文详细记录；同步更新仍在使用的项目说明、复现材料、证据地图和当前论文正文；把可以安全识别的旧稿、临时文件和精确重复文件集中到一个“待确认删除”目录，但不直接删除。

## 方案比较与选择

1. **仅修工作日志**：改动最小，但项目说明和论文会继续引用旧数字、旧脚本与旧口径。
2. **定向全量同步（采用）**：更新日志及所有仍在使用且受新实验影响的文档，历史材料保留追溯标签，重复与旧稿集中待删。完整性和风险之间最平衡。
3. **全项目重构**：重新组织全部脚本、输出和文档目录。范围过大，也可能破坏复现路径，本次不采用。

## 权威来源与优先级

1. 实验事实以当前脚本、CSV/NPZ/Markdown 汇总文件及可复算结果为准。
2. 当前论文 Markdown 以 `_manuscript.md` 为唯一源稿。
3. 当前 Word 正文以 `论文_TAP-Correct_PR中文正文.docx` 为唯一交付稿，并由 `_manuscript.md` 同步生成或校核。
4. `工作日志.md` 保留实验演进历史；被推翻的结果不删除，但必须明确标为“旧口径/已废弃/被后续实验取代”。
5. README、复现说明、协议、证据地图和执行计划只保留当前有效口径；历史草稿不再作为当前事实来源。

## 日志更新设计

- 将 2026-07-08 的 T1–T5 英文段落改成与前文一致的中文详细体例，统一包含：目的、协议/设置、执行命令或脚本、数据结果、解释边界、输出文件、对后续工作的影响。
- 合并 2026-07-09 重复的“一句话摘要”和后续详细记录，保留一份结构化版本。
- 补记 2026-07-09 晚间完成但日志遗漏的 Strawberry-DS 独立域实验，包括三种部署场景、关键数据、适用边界、脚本和产物。
- 补记 2026-07-10 的论文重构、正文生成和本次文档同步整理工作。
- 在日志顶部增加当前有效主线和“历史记录阅读规则”，避免读者把早期 V0/旧番茄结果误当成最终结论。

## 文档同步范围

必须核对并按当前实验更新：

- `README.md`
- `REPRODUCIBILITY.md`
- `FINAL_EXPERIMENT_PACKAGE.md`
- `V1_PROTOCOL.md`
- `执行计划.md`
- `证据地图.md`
- `审稿人问题清单.md`
- `论文重写大纲_PR版.md`
- `_manuscript.md`
- `论文_TAP-Correct_PR中文正文.docx`

仅作为历史材料保留、不再逐字更新的旧稿，移入待删除目录并在清单中注明原因。

## 待删除区设计

- 建立 `_待确认删除_2026-07-10/`。
- 子目录按 `01_精确重复`、`02_旧稿与备份`、`03_临时与锁文件`、`04_重复实验产物` 分类。
- 只移动满足以下条件的文件：字节级哈希完全相同；明确命名为备份/初稿/临时文件且已有确认的当前版本；实验输出与权威目录中的文件完全相同。
- 不移动原始数据、唯一实验产物、脚本、测试、人工标注审计链或无法确认用途的文件。
- 生成 `_待确认删除_2026-07-10/README_待删除清单.md`，记录原路径、现路径、分类、判定依据和保留的权威副本。
- 被 Word 占用的文件不强行移动；先在清单标记，待解除占用后再处理。

## 安全与一致性检查

- 修改前保存目标文件哈希与 Git 状态，避免覆盖用户现有未提交改动。
- 日志和文档中的关键数值必须能在对应 CSV/汇总文件中找到。
- 扫描旧番茄 `11.5% vs 17.3%`、旧 T2/T3/T4 脚本名、过时的 visual-only 主线、错误的 Proto-Adapter 弱基线表述等残留。
- Markdown 检查标题层级、表格列数、相对路径和 UTF-8 编码。
- Word 文档执行生成、渲染和页面级视觉检查；正文内容与 `_manuscript.md` 对齐。
- 运行与改动相关的测试和复现检查，最后再次审计 Git diff 与待删除清单。

## 完成标准

- 日志后半段恢复中文、详细、结构一致的实验记录，且遗漏实验已补齐。
- 当前文档之间的实验数据、方法主线、边界声明和文件路径一致。
- 当前正文 Markdown 与 Word 版本一致，Word 可正常打开且渲染无明显版式错误。
- 所有待删除候选均集中到一个文件夹并有可审计清单，项目中没有发生直接删除。
- 用户原有未提交代码与实验改动被保留。
